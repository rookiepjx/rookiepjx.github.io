            # canvas介绍

            Html5的canvas画布元素可以使用javaScript子网页上绘制图像

            canvas元素有三个属性，id,width,height。默认宽度300px,高度150px。

            canvas默认画布颜色是白色，初始状态下看不见，但是DOM中有canvas元素。

            # 兼容性

            canvas元素兼容ie9及以上版本浏览器。

            <canvas>
	你的浏览器版本太低，不兼容canvas，请升级！
</canvas>

            当浏览器支持canvas元素时，标签内的文字不会显示。否则会显示。

            # 绘制矩形

            fill是填充，stroke是描边，大多数情况下可以互换使用api。

            • **fillStyle 填充背景色 **

            • fillRect 绘制填充矩形 参数：x坐标，y坐标，宽度，高度

              绘制矩形必须在最后一步。

            			ctx.fillStyle = "rgb(200,0,0)";
			ctx.fillRect(10, 10, 300, 300);
			ctx.fillStyle = "rgb(0,0,200)"
			ctx.fillRect(30,30,300,300)

            • strokeRect 描边矩形 参数：x坐标，y坐标，宽度，高度
            • clearRect 清除矩形 参数：x坐标，y坐标，宽度，高度
            				ctx.fillStyle = "rgb(0,0,0)";
				ctx.fillRect(10, 10, 300, 300);
				ctx.clearRect(20, 20, 280, 280);
				ctx.strokeRect(50,50,220,220);

            # 路径绘制

            步骤：

            1. beginPath 创建路径
            2. moveTo 移动到起始点
            3. lineTo 连接路径
            4. closePath 闭合路径
            5. fill 或 stroke 绘制路径

            注意：如果最后lineTo到起点，可以不闭合路径。如果没有，closePath自动闭合。

            					ctx.beginPath();
					ctx.moveTo(50,50);
					ctx.lineTo(50,150);
					ctx.lineTo(100,100);
					ctx.fill();

            # 绘制圆形

            arc 绘制圆形

            圆形的组成：圆心（x,y),半径r,开始角度，结束角度，顺时针/逆时针

            开始角度：默认3点钟方向为0度

            顺时针/逆时针：默认false为顺时针

            			ctx.strokeStyle = "orange"
			ctx.lineWidth = 10
			ctx.arc(400,300,150,0,Math.PI*2,false)
			ctx.stroke();

            # CSS常用属性

            # 行内元素

            display:inline转换为行内元素

            特点：

            1. 内容决定元素所占位置
            2. 不能通过css改变宽高

            注意：

            • 行内元素的左右padding和margin是无效的

            常见行内元素：

            • a
            • span
            • em
            • strong

            # 块级元素

            display:block转换为块状元素

            特点：

            1. 元素独占一行
            2. 可以通过css改变宽高

            常见块级元素：

            • div
            • ol
            • ul
            • li
            • form
            • h1-h6
            • table

            # 行内块元素

            display:inline-block转换为行内块状元素

            特点:

            1. 内容决定大小
            2. 可以改变宽高

            常见行内块元素：

            • img
            • input
            • td
            • button

            # BFC

            概念：Block formatting contexts，块级格式化上下文。块级格式化上下文提供了一个环境，HTML元素在这个环境中按照一定规则进行布局。当前环境中的元素与外界环境相互独立，互不影响。

            触发条件：、

            • body根元素

            • float的值不能为none

            • overflow的值不能为visible

            • display的值为 inline-block，table-cell, table-caption中的任何一个

            • position的值不为relative和static

            BFC的特点：

            1. 内部的Box会在垂直方向上排列，垂直方向的距离由margin决定(属于同一个BFC的两个相邻Box的margin会发生重叠，与方向无关)

            2. 每个元素的margin box的左边， 与包含块border box的左边相接触(对于从左往右的格式化，否则相反)

            3. BFC的区域不会与float的元素区域重叠

            4. 计算BFC的高度时，浮动子元素也参与计算

            BFC作用：

            1. 解决浮动元素和非浮动元素重叠

            如果一个浮动元素后面跟着一个非浮动的元素，那么就会产生一个覆盖的现象，很多自适应的两栏布局就是这么做的

            <style>
    body {
        width: 300px;
        position: relative;
    }

    .aside {
        width: 100px;
        height: 150px;
        float: left;
        background: #f66;
    }

    .main {
        height: 200px;
        background: #fcc;
    }
</style>
<body>
    <div class="aside"></div>
    <div class="main"></div>
</body>

            由于两个box都处在同一个BFC中，都是以BFC边界为起点。如果两个box本身都具备BFC的话，会按顺序一个一个排列布局，现在.main并不具备BFC，根据BFC布局规则第2条:

            每个元素的margin box的左边， 与包含块border box的左边相接触

            虽然存在浮动的元素aslide，但main的左边依然会与包含块的左边相接触,根据BFC布局规则第3条:

            BFC的区域不会与float box重叠

            我们可以通过通过触发main生成BFC， 来实现自适应两栏布局

            .main {
    overflow: hidden;
}

            当触发main生成BFC后，这个新的BFC不会与浮动的aside重叠。因此会根据包含块的宽度，和aside的宽度，自动变窄。效果如下：

            1. 解决垂直margin重叠
            <style>
    p {
        color: #f55;
        background: #fcc;
        width: 200px;
        line-height: 100px;
        text-align:center;
        margin: 100px;
    }
</style>
<body>
    <p>Haha</p>
    <p>Hehe</p>
</body>

            两个p之间的距离为100px，发送了margin重叠。根据BFC布局规则第1条：

            内部的Box会在垂直方向上排列，垂直方向的距离由margin决定

            我们可以在p外面包裹一层容器，并触发该容器生成一个BFC。那么两个P便不属于同一个BFC，就不会发生margin重叠了。

            <style>
    .wrap {
        overflow: hidden;
    }
    p {
        color: #f55;
        background: #fcc;
        width: 200px;
        line-height: 100px;
        text-align:center;
        margin: 100px;
    }
</style>
<body>
    <p>Haha</p>
    <div class="wrap">
        <p>Hehe</p>
    </div>
</body>

            因此要解决margin重叠问题，只要让它们不在同一个BFC就行了

            1. 解决高度塌陷
            <style>
    .par {
        border: 5px solid #fcc;
        width: 300px;
    }

    .child {
        border: 5px solid #f66;
        width:100px;
        height: 100px;
        float: left;
    }
</style>
<body>
    <div class="par">
        <div class="child"></div>
        <div class="child"></div>
    </div>
</body>

            根据BFC布局规则第4条：

            计算BFC的高度时，浮动元素也参与计算

            为达到清除内部浮动，我们可以让par触发生成BFC，那么par在计算高度时，par内部的浮动元素child也会参与计算。

            .par{
  overflow:hidden
}

            # keyframes animation动画

            animatio是所有属性的简写。

            • animation-name 动画名称
            • animation-duration 动画持续时长
            • animation-timing-function 动画的速度曲线
            • animation-delay 画开始之前的延迟
            • animation-iteration-count 动画应该播放的次数
            • animation-direction 是否应该轮流反向播放动画
            • animation-fill-mode 规定当动画不播放时（当动画完成时，或当动画有一个延迟未开始播放时），要应用到元素的样式
            .box{
	width:200px;
	background:#fff;
	animation-name:animate1;
	animation-delay:1s;
	animation-duration:2s;
}
// from-to动画
@keykrames animate1 {
    //动画起始样式
	from{
		width:200px;
		top:0;
	}
    //动画结束样式
	to{
		width:600px;
		background:red;
		top:200px;
	}
}
// 百分比动画  盒子顺时针矩形运动
.box{
    background:#fff;
    width:200px;
    height:200px;
    top:0;
    left:0;
    animation:animate1 5s forwards ease-in-out;
}
@keykrames animate1 {
    //动画起始样式
	25%{
		top:0;
        left:300px;
	}
    //动画结束样式
	50%{
		top:300px;
        left:300px;
	}
    75%{
        top:300px;
        left:0;
    }
    100%{
        top:0;
        left:0;
    }
}

            # 行内非替换元素

            • 行内非替换元素如span、a、img使用margin、padding、border时，上下属性(top、bottom)无效，需要设置为display:inline-block

            # img底部白边

            • 由于图片默认display:inline和基线对齐vertical:baseline，在给父元素div背景色时，图片底部会多出白边。

              解决：给img设置display:block或者vertical-align:middle

            # text

            • text-decoration:line-through 定义从文本中间穿过的一条线

            • text-align:justify 文本两端对齐，平分文字间的空隙，但是不会对最后一行生效

            • text-align-last:justify 对文本最后一行进行两队对齐

            # 文本超出省略

            • 一行文本

              white-space:nowrap;
text-overflow:ellipsis;
overflow:hidden;

            • 多行文本

              text-overflow:ellipsis;
overflow:hidden;
display:-webkit-box;			//将对象作为弹性伸缩盒子模型显示。
-webkit-box-orient:vertical; 	//从上到下垂直排列子元素（设置伸缩盒子的子元素排列方式）
-webkit-line-clamp:2;			//这个属性不是css规范属性，需要组合上面两个属性，表示显示的行数。此处为2行

            # font属性

            • font-size:2em 使用em为单位时，继承父元素字体大小。eg:父元素字体10px,2em=2*10px

            • font-family 为防止操作系统内没有当前字体文件，设置多个字体，从左往右找到能使用的为止。英文字体写前面，中文字体写后面。

            • font-style 斜体

            1. italic 前提是font-family使用的字体支持斜体

            2. oblique 文本倾斜显示

            # line-height

            • 基线：与小写字母x最底部对齐的线

            • 行高严格的定义：两行文字基线之间的间距

            • line-height=文字高度+行距

            • 行距=line-height-文字高度，由于行距会上下等分，所以div内文本垂直居中：line-hieght=div的height

            # 元素隐藏

            • display:none 隐藏元素，不占空间

            • visibility:hidden 隐藏元素，依然占空间，元素点击事件失效

            • opacity:0 隐藏元素，透明，依然占空间，元素点击事件有效

            # min-width

            • 设置最小宽度，当浏览器宽度小于min-width，增加x滚动条

            # max-width

            • 当display：inline-block，宽度大于max-width，自动换行

            # margin

            margin塌陷

            margin值会取2个中的最大值，而非累加值

            • BFC:全称为 块格式化上下文 (Block Formatting Context),类似于结界，设置后不受外界干扰。
            • 行内非替换元素使用margin-top、margin-bottom无效，如span(需要使用display:inline-block)
            • 当使用margin-top：百分比时，百分比是相对于父元素的宽度

            # margin负值

            • 当元素不存在width或者width:auto的时候，负margin会增加元素的宽度。

              应用：

              1. 多个相同元素居中布局

              2. 左右、上下边框重叠，当边框为1px,可以设置margin-right：-1px解决左右边框重叠问题

            .sections {
            width: 990px;
            height: 300px;
            background-color: #f00;
            margin: 0 auto;
        }
        .wrap{

        }

        .section {
            width: 190px;
            height: 200px;
            background-color: blue;
            text-align: center;
            float: left;
            margin-right: 10px;
        }
        <div class="sections">
        <div class="wrap">
            <div class="section"></div>
            <div class="section"></div>
            <div class="section"></div>
            <div class="section"></div>
            <div class="section"></div>
        </div>
        </div>


            .sections {
            width: 1000px;
            height: 300px;
            background-color: #f00;
            margin: 0 auto;
        }
        .wrap{
            margin-right: -10px;  //解决元素居中问题
        }

        .section {
            width: 199px;
            border:1px solid #fff;
            height: 200px;
            background-color: blue;
            text-align: center;
            float: left;
            margin-right: -1px;  //解决边框重叠问题
        }

            # padding

            • 行内非替换元素使用padding-top、padding-bottom无效，虽然会将空间撑大，但是不会占据实际的空间，可能会与其他元素重叠。(使用display:inline-block)

            # border

            • 行内非替换元素的border-top、boder-bottom无效，虽然会将空间撑大，但是不会占据实际的空间，可能会与其他元素重叠。(使用display:inline-block)
            • 可以将元素高度、宽度设置0，边框宽度>0，画三角形

            # box-shadow

            • box-shadow: 水平偏移，垂直偏移，模糊范围，延伸范围，颜色
            • 设置多个方向的阴影时，使用逗号间隔，box-shadow：10px 10px 5px 5px,10px 10px 5px 10px
            • 当延伸范围为负数，可以防止其他方向出现阴影。

            # box-sizing

            • content-box:默认值，设置宽度和高度时，只是指定内容的高度和宽度
            • border-box:宽度和高度=内容+内边距+边框

            # position

            • relative 相对定位，相对于自己原来的位置

            • absolute 绝对定位，相对于最邻近的定位（非static）祖先元素定位。

              使用绝对定位后，元素脱离标准流，不占据空间，可以与其他元素重叠。

            • fixed 固定定位，使用后元素脱离标准流，相对于viewport视口定位

            • sticky 粘性布局，常用于吸顶效果，相当于relative+fixed，使用时增加定位，如top:100px,当距离顶部100px，变成固定定位。PC端浏览器兼容性不好，移动端兼容。

    top: 44px;
  }

            # 脱离标准流

            • 元素：position：absolute/fixed、float
            • 特点：多数元素的display属性会变成block,并且由于脱离标准流后无父元素，宽度和高度auto变成包裹的内容的高度和宽度。

            # float

            浮动的设计初衷是实现文字环绕效果。

            • 规则

              1. 浮动后脱离标准流，向浮动方向移动，直到父元素边界或其他浮动元素边界为止。

              ​ 定位过的元素会层叠在浮动元素之上

              1. 浮动元素不能与行内级内容重叠，**行内级元素（块级元素不会，会重叠）**会被浮动元素推挤出去，eg:行内级元素、inline-block元素、块级元素的文字内容

            1. 行内元素、inline-block元素浮动后，其顶部将与所在行的顶部对齐
            2. 浮动元素之间不能重叠，会被推出去。如果水平方向空间不足以显示浮动元素，浮动元素会向下移动
            3. 浮动元素的顶部不能超过包含块的顶部，也不能前一个浮动元素的顶部
            • 应用场景：解决行内级元素和inline-block元素布局会产生的水平间隙问题

            # inline-block布局缺陷

            • 行内级元素和inline-block元素布局会产生水平间隙

            • 使用float布局

            # 浮动高度坍塌

            • 产生原因：由于float元素会脱离标准流，所以不会向父元素汇报高度。
            • 解决方法（推荐使用伪元素）

            • 推荐方法：在需要浮动的父元素上增加伪元素

              .clear-fix::after{
	content:'';
	clear:both; //让元素的顶部低于所有浮动元素的顶部，从而实现清除浮动
	display:block;  //伪元素默认是行内元素
	height:0;
	visibility:hidden
}
<div class="container clear-fix">浮动内容</div>

            # flex布局

            • 概念：flex布局外层父元素为flex-container,里面的所有子元素为flex-item

              flex-item默认沿着main axis(主轴) 从main strat 开始往 main end 方向排布

            • display:flex flex-container作为块元素布局

              display:inline-flex flex-container作为行内元素布局

            • flex布局模型

            • flex container的CSS属性

              • flex-flow flex-direction和flex-wrap属性的缩写

              • flex-direction：决定main axis 主轴的方向

                1. row 默认值 从左向右
                2. row-reverse 从右到左
                3. column 从上到下
                4. column-reverse 从下到上

              • flex-wrap：决定flex container是单行还是多行

                1. nowrap 默认值 不换行，所有flex item在同一行显示
                2. wrap 多行
                3. wrap-reverse 多行，对比wrap,cross strat 与 cross end翻转

              • justify-content：决定flex-item 在main axis 主轴上的对齐方式

                1. flex-start 默认值 与main start对齐

                2. flex-end 与main end 对齐

                3. center 居中对齐

                4. space-between flex-item之间的距离相等，与main start、main end两端对齐

                5. space-evenly flex-item之间的距离相等，flex-item与main start、main end之间的距离等于flex-item之间的距离

                6. space-around flex-item之间的距离相等，flex-item与main start、main end

                  之间的距离是flex item 之间的距离的一半

              • align-items:决定了flex-item 在cross axis上的对齐方式

                1. normal:在弹性布局中，效果和stretch一样
              1. stretch:当flex-item在cross axis方向的size为auto时，会自动拉伸至填充flex container
              2. flex-start:与cross start对齐
              3. flex-end:与cross end 对齐
              4. center：居中对齐
              5. baseline:与基线对齐
              • align-content：决定了多行flex items 在 cross axis 上的对齐方式，用法与justify-content类似

            • flex item的CSS属性

              • flex

              • flex-grow

              • flex-shrink

              • flex-basis

              • order：决定了flex items的排布顺序

                1. 默认值为0

                2. 可以设置任意整数，值越小就排在越前面

              • align-self：可以通过align-self覆盖flex container设置的align-items

                1. auto 默认值，遵从flex container 的 align-items设置
                2. stretch、flex-start、flex-end、center、baseline,效果和align-items一致
