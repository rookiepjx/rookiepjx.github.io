              # TS

              Typescript相较于Javascript而言，最大的区别是：

              • 强类型
              • 接口
              • 泛型

              # 基础类型

              TS中包括JS中的所有类型，也新增了元组、枚举、任意、空、未知等其他类型。

              注意：在 .ts文件中声明并赋值，即使不声明数据类型也可以，数据类型默认是第一次赋值时的类型。TS会自动检测数据类型。

              • number

                let a : number  = 10;

              • string

                let a : string  = "hello";

              • boolean

                let a : boolean  = true;

              • 字面量

                限制变量的值就是该类型字面量的值。|代表或，联合类型

                let a : "male" | "female" ;
a = "male";
a = "female";

              • any

                任意类型，相当于关闭类型检测。不建议使用

                let a : any  = 10;
a = "hello";
a = true;

              • unknown

                未知类型，实际上是类型安全的any，不能直接赋值给其他变量

                let a : unknown;
a = 10;
a = "hello";
a = true;
let b : string = "hello"
b = a; // 报错,a是未知类型，b是string类型

// 解决：类型断言
// 1.变量 as 类型
// 2.<类型>变量
b = a as string;
b = <string>a;

              • void

                空值，即undefined类型

                function fn : void(){
	console.log(123);
}

              • never

                永远不会有值

                function fn : never(){
	throw new Error("error")
}

              • object

                由于JS中万物皆对象，所以直接指定object类型意义不大。

                常用{}来指定对象中包含的属性。?代表可选属性，可有可无。

                let a : {name:string,age?:number};
a = {name:"pjx",age:20};

                上面的方法会导致a对象只能有name和age属性，不可以有其他属性。

                // [propName:string]:any 表示任意类型的属性
let a : {name:string,[propName:string]:any};
a = {name:"pjx",age:20,vip:true}

              • array

                TS中对数组可以限制类型

                方法1：

                let a : string[];
a = ['a','b','c'];

                方法2：

                let a : Array<string>;
a = ['a','b','c'];

              • tuple

                元组类型，就是固定长度的数组。

                let a : [string,number];
a = ["hello",123];

              • enum

                枚举类型,需要先定义枚举类。

                // 枚举类
enum Gender{
    Male = 0,
    Female = 1
}
let a : {gender:Gender};
a = {gender:Gender.Male}

              补充：

              • &表示与类型，需要同时满足多种类型。
              let a : {name:string} & {age:number};
a = {name:"pjx",age:20}

              • 类型别名，使用type关键字定义类型常量。

                type mytype = 1 | 2 | 3;
let a : mytype;
a = 1;
a = 2;
a = 3;

              # 编译选项

              • 自动编译单个文件

                tsc 文件名 -w

              • 自动编译整个项目

                tsc

                需要配置TS，创建tsconfig.json文件

              • tsconfig.json文件配置选项

                • include 需要编译的文件
                • exclude 不需要编译的文件
                • extends 定义被继承的配置文件
                • files 指定被编译的文件列表，常用于需要编译的文件少的时候
                • compilerOptions TS编译器选项
                  • target 指定编译目标ES版本（es3、es5、es2015、es2016、es2017、es2018、es2019、es2020、esnext）
                  • modules 模块化规范 (commonjs、amd、system、umd、es6、es2015、es2020、esnext)
                  • lib 指定项目使用的库，通常浏览器端项目不需配置
                  • outDir 指定编译目标文件夹
                  • outFile 指定编译后的合并到一个文件
                  • allowJs 是否对JS文件进行编译，默认false
                  • checkJs 是否检查JS文件的数据类型，默认false
                  • removeComments 编译是否移除注释
                  • noEmit 不生产编译后的文件
                  • noEmitOnError 报错时不生成编译文件
                  • strict 下面所有严格检查的总开关，配置后下面都不需要配置
                  • alwaysStrict 设置编译后文件是否将开启严格模式，默认false
                  • noImplicitAny 不允许隐式any类型
                  • noImplicitThis 不允许类型不明确的this
                  • strictNullChecks 严格检查空值，当可能为空值时提示错误
                { 
    "include":[
        "./src/**/*"
    ],
    "exclude":[
        "./hello/**/*"
    ],
    "extends":[
        "./configs/base"
    ],
    "files":[
        "core.ts",
        "sys.ts",
        "type.ts"
    ],
    "compilerOptions":{
        "target":"ES6",
        "modules":"es2015",
        "lib":[],
        "outDir":"./dist",
        "outFile":"./dist/app.js",
        "allowJs":false,
        "checkJs":false,
        "removeComments":true,
        "noEmit":true,
        "noEmitOnError":true,
        "strict":true,
        "alwaysStrict":true,
        "noImplicitAny":true,
        "noImplicitThis":true,
        "strictNullChecks":true
    }
}

              # 类

              class Dog{
    // 类属性
    name : string;
    age: number;
    // 构造函数
    constructor(name : string,age : number){
        // this 代表当前类的实例对象
        this.name = name;
        this.age = age;
    }
    // 类方法
    bark(){
        console.log(this.name + "汪汪汪")
    }
}

              # 接口

              接口的作用就是规定类的结构。当一个类实现implements 接口时，需要具有接口中的所有属性和方法。

              注意：当有多个接口同名，会合并为一个接口。

              接口中的所有属性都没有实际值，所有方法都值抽象方法，不会有方法体的实现。

              interface myInterface{
    name:string;
    age:number;
    sayHello():void;
}

              // 类实现接口
class Dog implements myInterface{
	name:string;
    age:number;
    constructor(name:string,age:number){
        this.name = name;
        this.age = age;
    }
    sayHello(){
        console.log("hello")
    }
}

              # 属性封装

              类属性可以使用public、private、protected关键字指定类属性的访问权限。

              当外界访问、修改类属性时，dog.name、dog.name = "doggy"，会调用get和set方法。

              class Dog{
	public name : string;
    private address : string;
    protected age : number;
    get name(){
        return this.name;
    }
    set name(name:string){
        this.name = name;
    }
}

              # 泛型

              在定义类或者函数的时候，如果类型不确定，可以使用泛型。在调用的时候再是定具体类型。

              // 单类型
function fn<T>(a : T):T{
	return a;
}
// 多类型
function fn<T,K>(a : T,b : K):T{
    console.log(a);
	return b;
}
// 实现接口的泛型
interface myInterface{
    age : number;
}
function fn<T extends myInterface>(a : T) : number{
    return a.age;
}
// 类泛型
class Dog<T>{
    name : T;
    construcotr(name : T){
        this.name = name;
    }
}

const dog = new Dog<string>("doggy");
